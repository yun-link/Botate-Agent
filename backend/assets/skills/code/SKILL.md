### 工作流（严格执行）

**阶段一：项目规划（必须先执行）**
在编写任何代码之前，必须先进行完整的项目规划：

1. **需求理解与分析**
   - 明确用户的核心需求与业务目标
   - 识别功能边界与非功能需求（性能、安全、扩展性）

2. **技术方案设计**
   - 系统架构选型（架构图描述）
   - 技术栈选择（语言、框架、数据库、中间件）
   - 模块划分与职责定义
   - 数据流与接口设计

3. **输出项目规划**
   - 向用户呈现完整规划，无需确认后直接进入编码阶段
   - 规划必须包括：项目的完整树状文件、技术栈、架构设计。

**阶段二：代码实现**
- 按模块优先级逐步生成代码
- 每个模块包含：代码 + 关键注释 + 使用说明
- 确保代码符合最佳实践（SOLID原则、设计模式、安全规范）

**阶段三：验证与优化**
- 检查代码完整性、边界情况处理
- 提供测试思路和部署建议

### 编码规范

#### 代码质量要求
- **可读性**：清晰的命名、适当的注释、合理的代码结构
- **可维护性**：模块化设计、低耦合高内聚、单一职责原则
- **健壮性**：异常处理、输入验证、边界情况考虑
- **安全性**：防注入、防XSS、敏感信息保护、权限控制

#### 技术选型原则
- 优先选择成熟稳定的技术栈
- 根据团队熟悉度和社区活跃度选择框架
- 考虑部署成本和运维复杂度
- 为扩展性预留空间，避免过度设计
